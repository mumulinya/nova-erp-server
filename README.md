# 🚀 Nova ERP —— 新一代智能“业财仓一体化”管理系统

**严肃声明：本项目为个人毕业设计/开源学习项目，现在、未来都不会有闭源商业版本，所有代码 100% 全部开源！**

## 📖 1. 项目简介

传统 ERP 系统往往面临着授权昂贵、操作臃肿、学习曲线极高的痛点。中小型商贸及物流企业迫切需要一款**轻量、敏捷、且具备极高业务拓展性**的信息化平台。

**Nova ERP** 应运而生。本项目采用业界最成熟的 **Spring Boot 3 + Vue 3 + MyBatis-Plus** 前后端分离架构，深度融入 **SaaS 多租户** 隔离思想。我们打破了传统系统仅仅作为“数据增删改查记录仪”的桎梏，**开创性地集成了 Spring AI 与大语言模型（如 GLM-4）**，实现了从底层数据流转到顶层 AI 智能决策的全方位赋能。

系统内不仅包含采购、销售、库存等常规模块，更深度研发了**准 WMS 级仓储盘点与调拨**、**细颗粒度财务流水联动**、以及**正逆向退换货闭环**。

## 🌟 2. 核心特性与独创亮点

### 🤖 2.1 全矩阵 AI 智能分析引擎 (独家特性)
区别于只接入单一对话框的伪 AI 项目，Nova ERP 将 AI 能力拆解并下沉到每一个具体的业务环节，真正让大模型为老板和员工打工：
- **AI 采购进货建议 (`ErpPurchaseAiSuggest`)**：通过长周期历史均价、每日动销率与当前库存红线交汇分析，大模型主动计算出本月建议补货清单与合理买入区间。
- **AI 销售趋势诊断 (`ErpSaleAiAnalysis`)**：基于季节、订单量、客单价等维度的异动，让大语言模型化身为“营销总监”，给出促销或客户维系的诊断意见。
- **AI 库存积压及物流止损 (`ErpStockAiSuggest`)**：精准定位库龄超长、运输费用严重倒挂的“死库存”，出具打折清仓或更换指定承运商的智能报告。

### 📦 2.2 准 WMS 级别专业仓管 (Stock & Warehouse)
对糊涂账说不，建立银行级精密台账：
- **库存盘点 (StockCheck)**：定期对实体仓库发起盘点，系统自动对冲账面与实物差值，一键生成盘盈/盘亏流水。
- **仓库调拨 (StockMove)**：支持区域仓到总仓的无损调拨，严格保障调出扣减与调入增加的强一致性（原子操作）。
- **防篡改出入库流水 (StockRecord)**：所有库存变动禁止执行直接 UPDATE 语句，强制走事务拦截并生成带有业务单据来源的详细台账。

### 🔄 2.3 正逆向业财全生命周期闭环 (Purchase, Sale & Finance)
- **严格的正向防超卖出库**：任何销售出库动作在底层由乐观锁严防并发漏洞，彻底杜绝高并发超卖。
- **完整的逆向物流退换货**：完美支持 `采购退货单` 与 `销售退货单`，冲正流程自动关联实物库存退库。
- **业财一体，资金随动物流**：每一笔出库都会自动挂起一笔应收收款单 (`ErpFinanceReceipt`)，每一笔入库自动挂起一笔应付付款单 (`ErpFinancePayment`)，财务人员再也无需使用 Excel 记账。

### 🛡️ 2.4 银行级 SaaS 数据隔离底座
- 基于 `TenantContextWebFilter` 配合 MyBatis-Plus `TenantLineInnerInterceptor`，开发者完全**不需要在业务层硬编码**租户条件。
- 哪怕是高并发的多线程操作环境，`TransmittableThreadLocal` 依然能完美传递租户令牌，构建了牢不可破的数据防火墙，让同一个平台支持千万家企业同时使用。

## 📂 4. 核心系统目录结构解读

为了方便开源学习，这里梳理出后端的重点目录结构：
```text
nova-erp-server
├── yudao-framework/            # 全局通用底层框架扩展（包含多租户插件、安全认证等）
└── yudao-module-erp/           # 🌟 核心业务模块
    └── src/main/java/cn/iocoder/yudao/module/erp/
        ├── controller/admin/   # RESTful 接口层
        │   ├── ai/             # 智能分析引擎 (采购/销售/库存大模型对接)
        │   ├── finance/        # 财务流转中心 (账户、收付款)
        │   ├── logistics/      # 物流轨迹追踪中心
        │   ├── product/        # 商品 SKU 与类目管理
        │   ├── purchase/       # 采购入库及退货逆向流
        │   ├── sale/           # 销售出库及退换逆向流
        │   ├── statistics/     # Redis 加速的数据大屏展示报表
        │   └── stock/          # WMS级 仓储管理 (调拨、盘点、出入库台账)
        ├── service/            # 核心业务逻辑层 (事务控制集中在此)
        ├── dal/                # 数据访问层 (MyBatis Mapper 接口)
        └── framework/          # 针对 ERP 业务的个性化租户上下文配置
```

## 🛠️ 6. 本地部署指南 (Quick Start)

### 环境要求
- **JDK**： 17
- **Node.js**： 18.x
- **MySQL**： 8.0.x
- **Redis**： 5.0+

### 第一步：克隆项目与数据库准备
1. 通过 Git 将前后端代码克隆至本地。
2. 打开 MySQL 客户端，创建数据库 `nova_erp`。
3. 执行项目根目录下的 `sql/nova-erp.sql` （如有）以初始化数据与权限表。

### 第二步：启动后端服务
1. 推荐使用 IntelliJ IDEA 打开后端工程 `nova-erp-server`。
2. 找到 `application-dev.yaml`，修改如下配置项：
   - 数据库连接信息 (`username`, `password`, `url`)。
   - Redis 连接信息。
   - (可选) 如果需要测试大模型，在 AI 配置中填入您申请的有效 API KEY。
3. 运行主启动类 `NovaErpApplication.java`。

### 第三步：启动前端管理控制台
1. 打开终端，进入前端工程目录。
2. 执行依赖安装：
   ```bash
   npm install --registry=https://registry.npmmirror.com
   ```
3. 启动开发服务器：
   ```bash
   npm run dev
   ```
4. 在浏览器中打开提示的本地地址（如 `http://localhost:80`），使用默认超管账号登录。
