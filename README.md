# Nova ERP —— 新一代智能"业财仓物流一体化"管理系统

**严肃声明：本项目为个人毕业设计/开源学习项目，现在、未来都不会有闭源商业版本，所有代码 100% 全部开源！**

## 1. 项目简介

传统 ERP 系统往往面临着授权昂贵、操作臃肿、学习曲线极高的痛点。中小型商贸及物流企业迫切需要一款**轻量、敏捷、且具备极高业务拓展性**的信息化平台。

**Nova ERP** 应运而生。本项目采用业界最成熟的 **Spring Boot 3 + Vue 3 + MyBatis-Plus** 前后端分离架构，深度融入 **SaaS 多租户** 隔离思想。我们打破了传统系统仅仅作为"数据增删改查记录仪"的桎梏，**开创性地集成了 Spring AI 与大语言模型（如 GLM-4）**，实现了从底层数据流转到顶层 AI 智能决策的全方位赋能。

系统内不仅包含采购、销售、库存等常规模块，更深度研发了**准 WMS 级仓储盘点与调拨**、**细颗粒度财务流水联动**、**完整的物流调度与 AI 智能派车**、以及**正逆向退换货闭环**，真正实现了"业财仓物流"四大业务域的一体化贯通。

## 2. 核心特性与独创亮点

### 2.1 AI 智能矩阵引擎（独家特性）

区别于只接入单一对话框的伪 AI 项目，Nova ERP 将 AI 能力拆解并下沉到**五个**具体的业务环节，真正让大模型为老板和员工打工：

| AI 服务 | 入口 | 核心能力 |
| :--- | :--- | :--- |
| **AI 采购进货建议** | `POST /erp/purchase/ai-suggest/generate` | 通过长周期历史均价、每日动销率与当前库存红线交汇分析，主动计算本月建议补货清单与合理买入区间 |
| **AI 销售趋势诊断** | `POST /erp/sale/ai-analysis/generate` | 基于季节、订单量、客单价等维度的异动，让大语言模型化身为"营销总监"，给出促销或客户维系的诊断意见 |
| **AI 库存积压预警** | `POST /erp/stock/ai-suggest/generate` | 精准定位库龄超长、运输费用严重倒挂的"死库存"，出具打折清仓或更换承运商的智能报告 |
| **AI 供应链全景分析** | `GET /erp/ai-analysis/stream` | 实时汇聚大屏 KPI、库存健康度、物流状态、30 天趋势数据，以 SSE 流式方式输出供应链全局诊断报告 |
| **AI 物流智能调度** | `POST /erp/logistics-order/ai-dispatch` | 自动从候选车辆中选取最优匹配车辆，并生成包含距离、时长、注意事项的完整路线规划方案 |

所有 AI 服务均通过 Spring AI 的 `ChatClient` 对接大模型，支持通过 YAML 配置一键切换底层模型（GLM-4 / GPT-4 / Claude 等），业务代码零改动。

### 2.2 物流调度与运输管理（核心差异化模块）

系统围绕**"运单 - 车辆 - 路线 - 费用"**四大实体构建了完整的物流调度体系：

- **运单管理**：支持"待调度 → 运输中 → 已送达 → 异常"四状态流转，关联销售出库单，出库时间门控防止仓库未发货即创建运单。
- **车队管理**：车辆状态（空闲/运输中/维修中）随运单生命周期自动流转，避免同一车辆被重复分配。
- **AI 智能派车**：一键触发大模型执行"智能选车 + 路线规划"，自动解析距离和时长数值，支持范围值取平均、天/小时自动换算。
- **路线规划**：AI 生成的路线以 Markdown 格式完整保存建议内容，便于前端渲染和审计追溯。
- **成本核算**：按运单归集运输费、燃油费、过路费等明细，为物流成本趋势分析提供数据基础。

### 2.3 准 WMS 级别专业仓储（Stock & Warehouse）

对糊涂账说不，建立银行级精密台账：

- **库存盘点**：定期对实体仓库发起盘点，系统自动对冲账面与实物差值，一键生成盘盈/盘亏流水。
- **仓库调拨**：支持区域仓到总仓的无损调拨，严格保障调出扣减与调入增加的强一致性（原子操作）。
- **防篡改出入库流水**：所有库存变动禁止执行直接 UPDATE 语句，强制走事务拦截并生成带有业务单据来源的详细台账。

### 2.4 正逆向业财全生命周期闭环（Purchase, Sale & Finance）

- **严格的正向防超卖出库**：任何销售出库动作在底层由悲观锁严防并发漏洞，彻底杜绝高并发超卖。
- **完整的逆向物流退换货**：完美支持采购退货单与销售退货单，冲正流程自动关联实物库存退库。
- **业财一体，资金随动物流**：每一笔出库都会自动挂起一笔应收收款单，每一笔入库自动挂起一笔应付付款单，财务人员再也无需使用 Excel 记账。

### 2.5 银行级 SaaS 数据隔离底座

- 基于 `TenantContextWebFilter` 配合 MyBatis-Plus `TenantLineInnerInterceptor`，开发者完全不需要在业务层硬编码租户条件。
- `TransmittableThreadLocal` 确保高并发多线程环境下租户令牌的完美传递，构建牢不可破的数据防火墙。

### 2.6 数据大屏与统计报表

- **供应链全景大屏**：汇集 KPI 核心指标、30 天销售/采购趋势、库存健康度、物流运单状态分布、物流成本趋势六大维度。
- **四大业务报表**：采购汇总、销售汇总、库存预警、物流成本明细，支持时间区间筛选和 Excel 一键导出。
- **Redis 高速缓存**：大屏数据经 Redis 方法级缓存加速，P99 响应从 2500ms 优化至 15ms。

## 6. 核心业务流程

### 6.1 采购入库跨模块联动

```
采购员提交入库单
  → 校验采购订单已审批
  → 校验结算账户有效
  → Redis 自动生成入库单号 (RKD2024050100001)
  → 写入入库单 + 明细行
  → 回写采购订单的已入库数量
  → 审核通过后：遍历商品明细 → 逐行增加库存 → 写入流水
```

### 6.2 销售出库防超卖与业财联动

```
销售人员提交出库单
  → 库存悲观锁扣减 (UPDATE SET count=count-? WHERE count>=?)
  → 库存不足则抛出异常回滚，杜绝超卖
  → 扣减成功后写入库存流水 (SALE_OUT)
  → 可创建物流运单 → AI 智能派车
  → 可挂账应收收款单 → 财务审批入账
```

### 6.3 AI 智能物流调度

```
调度员点击"AI调度"按钮
  → 解析运单关联的销售出库单 → 追溯真实发货仓库地址
  → AI 从空闲车辆中选取最优车辆（考虑货物重量/体积/车牌归属地）
  → AI 生成路线规划（高速/国道选择、途经节点、注意事项）
  → 正则解析距离（支持范围值取平均）和时长（支持天→小时换算）
  → 自动创建路线记录 + 绑定车辆和路线到运单
```

## 7. 本地部署指南 (Quick Start)

### 环境要求

| 环境 | 版本要求 |
| :--- | :--- |
| JDK | 17 |
| Node.js | 18.x |
| MySQL | 8.0+ |
| Redis | 5.0+ |

### 第一步：克隆项目与数据库准备

1. 通过 Git 将前后端代码克隆至本地。
2. 打开 MySQL 客户端，创建数据库 `nova_erp`。
3. 执行项目 `sql/mysql/` 目录下的 SQL 脚本以初始化表结构与基础数据。

### 第二步：启动后端服务

1. 推荐使用 IntelliJ IDEA 打开后端工程 `nova-erp-server`。
2. 找到 `application-local.yaml`，修改数据库连接（`username`、`password`、`url`）和 Redis 连接信息。
3. （可选）如需测试 AI 功能，在 `application.yaml` 的 AI 配置中填入有效的 API Key。
4. 运行主启动类 `NovaErpApplication.java`。

### 第三步：启动前端管理控制台

1. 打开终端，进入前端工程目录。
2. 安装依赖并启动：
   ```bash
   npm install --registry=https://registry.npmmirror.com
   npm run dev
   ```
3. 浏览器打开提示的本地地址（如 `http://localhost:80`），使用默认超管账号登录。
